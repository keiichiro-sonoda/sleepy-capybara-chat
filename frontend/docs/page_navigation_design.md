# AIチャットアプリケーション - ページ遷移設計書

## 概要

このドキュメントでは、AIチャットアプリケーションのページ遷移フローと各ページの役割について説明します。

## ページ構成

### 1. ランディングページ
- **パス**: `/`
- **役割**: アプリケーションの紹介、ユーザー登録・ログインへの誘導
- **遷移先**:
  - ログイン画面 (`/auth/login`)
  - 登録画面 (`/auth/register`)
  - デモ画面 (`/demo`)

### 2. 認証関連ページ

#### 2.1 ログイン
- **パス**: `/auth/login`
- **役割**: 登録済みユーザーの認証
- **遷移先**:
  - チャットホーム (`/chat`)
  - パスワードリセット (`/auth/reset-password`)
  - 登録ページ (`/auth/register`)

#### 2.2 登録ページ
- **パス**: `/auth/register`
- **役割**: 新規ユーザー登録
- **遷移先**:
  - メール確認案内 (`/auth/verify-email-sent`)
  - ログイン (`/auth/login`)

#### 2.3 メール確認案内
- **パス**: `/auth/verify-email-sent`
- **役割**: メール確認の案内表示
- **遷移先**:
  - ログイン (`/auth/login`)

#### 2.4 メール確認
- **パス**: `/auth/verify-email`
- **役割**: メールリンクからのアクセスで、ユーザーのメールアドレスを確認
- **パラメータ**: `token` (認証トークン)
- **遷移先**:
  - 確認成功時: ログイン (`/auth/login`)
  - 確認失敗時: エラーページ (`/error`)

#### 2.5 パスワードリセット要求
- **パス**: `/auth/reset-password`
- **役割**: パスワードリセットのメール送信リクエスト
- **遷移先**:
  - リセットメール送信完了 (`/auth/reset-password-sent`)

#### 2.6 パスワードリセットメール送信完了
- **パス**: `/auth/reset-password-sent`
- **役割**: パスワードリセットメールの送信完了を通知
- **遷移先**:
  - ログイン (`/auth/login`)

#### 2.7 新パスワード設定
- **パス**: `/auth/new-password`
- **役割**: 新しいパスワードの設定
- **パラメータ**: `token` (リセットトークン)
- **遷移先**:
  - 設定成功時: ログイン (`/auth/login`)
  - 設定失敗時: エラーページ (`/error`)

### 3. チャット関連ページ

#### 3.1 チャットホーム
- **パス**: `/chat`
- **役割**: チャット一覧と新規チャット作成
- **遷移先**:
  - 個別チャット (`/chat/[chatId]`)
  - 設定 (`/settings`)
  - プロフィール (`/profile`)

#### 3.2 個別チャット
- **パス**: `/chat/[chatId]`
- **役割**: 特定のチャットルームでのAIとの対話
- **パラメータ**: `chatId` (チャットの一意識別子)
- **遷移先**:
  - チャットホーム (`/chat`)
  - チャット設定 (`/chat/[chatId]/settings`)

#### 3.3 チャット設定
- **パス**: `/chat/[chatId]/settings`
- **役割**: 特定チャットの設定変更
- **パラメータ**: `chatId` (チャットの一意識別子)
- **遷移先**:
  - 個別チャット (`/chat/[chatId]`)

### 4. ユーザー関連ページ

#### 4.1 プロフィール
- **パス**: `/profile`
- **役割**: ユーザープロフィールの表示・編集
- **遷移先**:
  - チャットホーム (`/chat`)
  - アカウント設定 (`/settings/account`)

#### 4.2 設定
- **パス**: `/settings`
- **役割**: アプリケーション全般の設定
- **遷移先**:
  - アカウント設定 (`/settings/account`)
  - 通知設定 (`/settings/notifications`)
  - プライバシー設定 (`/settings/privacy`)
  - チャットホーム (`/chat`)

### 5. その他

#### 5.1 デモページ
- **パス**: `/demo`
- **役割**: 未登録ユーザー向けのデモチャット体験
- **遷移先**:
  - ランディングページ (`/`)
  - 登録ページ (`/auth/register`)

#### 5.2 エラーページ
- **パス**: `/error`
- **役割**: エラー情報の表示
- **遷移先**:
  - ランディングページ (`/`)
  - 直前のページ (ブラウザバック)

## 認証フロー

1. **新規ユーザー登録フロー**:
   - ランディングページ → 登録ページ → メール確認案内
   - ユーザーはメール内のリンクをクリック → メール確認 → ログイン → チャットホーム

2. **ログインフロー**:
   - ランディングページ → ログイン → チャットホーム

3. **パスワードリセットフロー**:
   - ログイン → パスワードリセット要求 → リセットメール送信完了
   - ユーザーはメール内のリンクをクリック → 新パスワード設定 → ログイン

## 遷移制約

1. **認証要件**:
   - `/chat/*` および `/settings/*`, `/profile` へのアクセスには認証が必要
   - 未認証時のアクセスはログイン画面へリダイレクト

2. **認証済みリダイレクト**:
   - 認証済みユーザーが `/auth/*` にアクセスした場合、チャットホームへリダイレクト

## モバイル対応

- すべてのページはレスポンシブデザインで実装
- モバイル特有の遷移パターン:
  - チャット一覧とチャット詳細が分離表示
  - 設定メニューがハンバーガーメニューに統合

## 実装上の注意点

1. Next.jsのApp Routerを使用したルーティング実装
2. ミドルウェアによる認証状態チェックと適切なリダイレクト
3. レイアウトグループを活用した共通ナビゲーション
4. SSR/CSR適切な使い分けによるパフォーマンス最適化
